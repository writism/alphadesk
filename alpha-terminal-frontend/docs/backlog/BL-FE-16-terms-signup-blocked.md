# BL-FE-16

**Backlog Type**
Behavior Backlog

**Backlog Title**
애플리케이션이 약관 동의 후 회원가입 단계로 정상 진행된다

**배경**
현재 `/terms` 페이지에서 필수 약관을 모두 동의하고 `동의하고 계속하기`를 눌러도
회원가입 단계로 진행되지 않는 것처럼 보이는 문제가 있다.
신규 사용자 경로에서 `email` 쿼리 값이 빈 문자열로 전달되면 `/account/signup` 페이지가
프로필 누락으로 판단하여 다시 `/terms`로 리다이렉트하고,
프론트 회원가입 요청도 실제 백엔드 엔드포인트와 일치하지 않아 다음 단계가 막힌다.

**Success Criteria**
- `/terms?nickname=...&email=` 상태에서도 필수 약관 동의 후 회원가입 페이지로 이동할 수 있다
- `/account/signup` 페이지는 `nickname`이 있으면 표시되고, `email`이 비어 있으면 직접 입력할 수 있다
- `email`이 이미 전달된 경우에는 기존처럼 읽기 전용으로 표시된다
- 회원가입 요청이 실제 백엔드 엔드포인트와 일치하는 경로로 전송된다
- 회원가입 성공 시 메인 페이지(`/`)로 이동한다
- 더 이상 약관 페이지와 회원가입 페이지 사이에서 무한 리다이렉트가 발생하지 않는다

**Todo**
1. `/account/signup` 페이지의 진입 조건을 `nickname` 중심으로 재정의한다
2. `email` 값이 비어 있을 때는 입력 가능한 필드로 처리한다
3. `email` 값이 있는 경우에는 읽기 전용 표시를 유지한다
4. 회원가입 API 호출 경로를 백엔드 실제 엔드포인트에 맞게 수정한다
5. 약관 동의 후 회원가입 완료까지 흐름을 점검한다
