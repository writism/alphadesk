# BL-FE-34

**Backlog Type**  
UX / IA / Bugfix

**Backlog Title**  
히트맵 기준일 설명·공통 범례·대시보드 히트맵 노출

---

## 1. 배경 / 문제

1. **기준일 문구**  
   `전일 종가 대비 · YYYY-MM-DD`만으로는 “그 날짜가 무엇을 뜻하는지”(히트맵에 포함된 **마지막 거래일**)가 바로 이해되지 않는다.

2. **범례 중복**  
   카드마다 하락/보합/상승 범례가 반복되어 시각적 노이즈가 크다. **페이지(섹션)당 한 번**이면 충분하다.

3. **대시보드 히트맵 미노출**  
   파이프라인 실행 중(`running`)일 때 요약 섹션이 **진행 패널만** 보여 기존 요약 카드·히트맵이 통째로 사라진다. 또한 심볼 대소문자 불일치 시 `bySymbol` 조회가 실패할 수 있다.

---

## 2. 목표 (Success Criteria)

| ID | 기준 |
|----|------|
| SC-1 | 기준일은 **“히트맵에 포함된 마지막 거래일”**임을 짧은 문구·(선택) `title`로 설명한다. |
| SC-2 | 범례는 **관심종목 목록 블록·대시보드 AI 요약 블록** 각각 상단에 **최대 1회**만 둔다. 카드 내부 범례는 제거한다. |
| SC-3 | 대시보드에서 `running`이어도 **이미 로드된 요약이 있으면** 카드(히트맵 포함)를 진행 패널 아래에 계속 보여 준다. |
| SC-4 | 히트맵 `bySymbol` 조회 키는 **대문자 정규화**로 요약/관심종목 심볼과 맞춘다. |
| SC-5 | 카드별 기준일은 **해당 종목 시리즈의 마지막 거래일**을 쓰고, 배치 응답의 전역 `as_of`에 묶지 않는다. |

## 4. 후속 (제품 피드백 반영)

- 범례 끝 문구는 **「색 농도 = 변동 강도」** 만 유지(감성 배지 대비 문구 제거).
- **최근 AI 분석 로그** 카드에도 요약과 동일 배치 히트맵 표시; 심볼 집합은 `요약 + 로그` 합집합으로 한 번만 fetch.
