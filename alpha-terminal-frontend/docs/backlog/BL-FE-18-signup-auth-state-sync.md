# BL-FE-18

**Backlog Type**
Behavior Backlog

**Backlog Title**
애플리케이션이 회원가입 성공 후 인증 상태를 즉시 반영한다

**배경**
현재 신규 사용자가 약관 동의와 회원가입을 완료한 뒤 메인 페이지로 이동하면,
회원가입 응답으로 `nickname`, `email`, `account_id` 쿠키는 정상 세팅되더라도
프론트 전역 인증 상태가 여전히 `PENDING_TERMS`로 남아 있을 수 있다.
이 경우 Navbar가 로그인/로그아웃 버튼을 렌더링하지 않아
사용자 입장에서는 로그인 상태를 확인하거나 로그아웃할 수 없는 상태가 된다.

**Success Criteria**
- 회원가입 성공 직후 전역 인증 상태가 `AUTHENTICATED`로 갱신된다
- 회원가입 후 메인 페이지에서 Navbar의 로그인/로그아웃 버튼이 정상적으로 표시된다
- 인증된 상태에서는 Dashboard, Watchlist 메뉴가 정상적으로 노출된다
- 새로고침 없이도 회원가입 직후 UI가 올바른 인증 상태를 반영한다
- 기존 로그인 및 로그아웃 흐름에는 회귀가 없다

**Todo**
1. 회원가입 성공 시 쿠키 기반 인증 정보를 다시 읽어 auth state를 갱신한다
2. 회원가입 성공 후 이동 시점에 `AUTHENTICATED` 상태가 반영되도록 구현한다
3. Navbar가 회원가입 직후에도 로그인 상태를 올바르게 표시하는지 점검한다
