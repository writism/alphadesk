# BL-FE-30

**Backlog Type**  
UX / Data Visualization Backlog

**Backlog Title**  
종목 카드 내 일별 등락 히트맵(깃허브 기여도 스타일) — 저빈도 API·고정보 UI

> **구현 상태:** 재사용 컴포넌트·배치 훅·대시보드/관심종목 연동·한 줄+wrap 레이아웃은 구현됨. 데이터 소스 전환·페이지 간 캐시는 **BL-FE-31** 참고.

---

## 1. 배경 / 문제

- 실시간 시세(초·분 단위 갱신)는 **외부 API 소비·복잡도**가 크다.
- 대신 **최근 수주~약 1달**의 **일별 등락**을 **미니 격자**(주 열 × 요일 행, 깃허브 contribution graph 유사)로 보여 주면, **적은 데이터로 추세·변동성**을 한눈에 전달할 수 있다.
- **BL-BE-13**에서 **배치·캐시·슬림 페이로드** API를 제공한다는 전제에서, 프론트는 **호출 횟수·페인트 비용**을 최소화한다.

---

## 2. 목표 (Success Criteria)

| ID | 기준 |
|----|------|
| SC-1 | `StockSummaryCard`(대시보드 요약)·관심종목 카드(또는 행)에 **히트맵 슬롯**이 들어갈 수 있는 **재사용 컴포넌트**가 있다. |
| SC-2 | 레이아웃은 **깃허브 기여도 그래프**와 같이 **한 열 = 1주**, **세로 = 요일(7행)**; 표시 기간은 기본 **최근 6주** 등 합의 값. |
| SC-3 | 셀 색은 **상승/하락/보합** + **강도 단계**(백엔드 bucket)를 매핑; **국내 관습(상승 빨강·하락 파랑)** vs 글로벌 색상은 **토큰/상수 한 곳**에서 전환 가능. |
| SC-4 | 동일 페이지에 카드가 여러 개일 때, **한 번의 배치 요청**으로 모든 심볼 데이터를 가져오고 **React Query/SWR** 등으로 **staleTime**을 길게(예: 5~30분) 잡아 **재요청을 최소화**한다. |
| SC-5 | 접근성: 색만에 의존하지 않고 **요약 텍스트** 또는 `aria-label`(예: “최근 6주 상승 12일 하락 8일”)을 제공한다. |
| SC-6 | 데이터 없음·부분 실패 시 **회색/빈 격자 + 짧은 안내**로 레이아웃이 무너지지 않는다. |
| SC-7 | **직관 보강:** 셀 `title`/툴팁은 개발자용 `bucket N`이 아니라 **전일 대비 등락 분위(한글)** 로 표현한다. |
| SC-8 | **범례:** 빨강/파랑이 **등락(수익률 분위)** 임을 한 줄로 표시하고, **감성 배지 색과 혼동하지 말 것** 안내를 둔다. |
| SC-9 | **한국장:** `market`이 KRX 계열이면 **전일 종가·지연 공시** 가능성을 짧은 각주로 표시한다. |
| SC-10 | **카드 IA:** `StockSummaryCard`에서 **요약 본문을 히트맵보다 위**에 두어 먼저 읽히게 한다. |

## 3. 무료 트래픽을 줄이는 프론트 전략

1. **배치만 사용**  
   - 카드 마운트마다 `GET` 난사 금지. 페이지 단위 `symbols[]` 수집 후 **단일 쿼리 키**로 fetch.
2. **staleTime / cacheTime**  
   - 일봉 기반이면 **수 분~수십 분** 동안 fresh로 간주; 탭 포커스 refetch는 끄거나 완화.
3. **겹치는 페이지**  
   - 대시보드 ↔ 관심종목 동일 심볼이면 **동일 캐시 키** 재사용(심볼 정규화).
4. **스켈레톤**  
   - 로딩 중 격자 **고정 높이** 유지로 CLS 방지(불필요한 reflow 감소 = 체감 품질).

## 4. UI 스펙 (초안)

### 4.1 치수

- 셀: 약 `10×10px` ~ `12×12px`, `gap` 2~3px(모바일에서 터치 타겟이 아니므로 **장식 요소**로만 취급).
- 전체 블록 최대 너비: 카드 본문 너비 내(요약 카드 전체 폭과 정렬).

### 4.2 색 매핑 (예시)

| bucket | 의미 | 다크모드 고려 |
|--------|------|----------------|
| `+2` | 큰 상승 | 진한 빨강 |
| `+1` | 작은 상승 | 연한 빨강 |
| `0` | 보합/미미 | 회색 |
| `-1` | 작은 하락 | 연한 파랑 |
| `-2` | 큰 하락 | 진한 파랑 |

*사용자가 이전에 “파랑/빨강”을 언급했으나, **국내 UX**와 충돌 시 **상수 스왑**으로 조정.*

### 4.3 보조 정보 (추가 API 없음)

- BE가 주는 `summary.up/down/flat` 및 `as_of`를 격자 **아래 또는 옆**에 `text-xs`로 표시.

---

## 5. 데이터 계약 (BL-BE-13과 맞출 것)

- 입력: `symbol`, `market`(카드에 이미 있으면 전달).
- 응답: `series` = 거래일별 `[date, bucket]` + `summary` + `as_of`.
- 프론트 책임: `series` → **주×요일 격자** 좌표 매핑(휴장은 빈 칸).
