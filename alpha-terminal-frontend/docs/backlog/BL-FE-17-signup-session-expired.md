# BL-FE-17

**Backlog Type**
Behavior Backlog

**Backlog Title**
애플리케이션이 회원가입 임시 세션 만료를 정상 처리한다

**배경**
신규 사용자가 카카오 로그인 후 약관 동의와 회원가입 입력을 진행하는 동안
가입용 `temp_token` 세션이 만료되면 `/account/register` 요청이 `401 Unauthorized`로 실패한다.
현재는 사용자가 원인을 알기 어려운 상태로 로그인 페이지로 돌아가거나 콘솔 에러만 보게 된다.
임시 세션 유지 시간을 현실적인 수준으로 늘리고,
만료 시에도 사용자가 다시 로그인해야 한다는 안내를 명확히 보여 줄 필요가 있다.

**Success Criteria**
- 약관 확인과 회원가입 입력에 필요한 시간 동안 가입용 임시 세션이 충분히 유지된다
- 가입용 임시 세션이 만료되면 사용자는 원인을 이해할 수 있는 안내를 본다
- 회원가입 요청이 `401`로 실패할 때 로그인 페이지에서 재로그인 필요 메시지가 표시된다
- 로그인 재시작 후 신규 회원 흐름을 다시 정상 진행할 수 있다
- 기존 회원 로그인 흐름에는 회귀가 없다

**Todo**
1. 신규 회원 가입용 `temp_token`과 관련 쿠키의 TTL을 조정한다
2. 프론트 회원가입 요청에서 `401` 응답을 가입 세션 만료로 처리한다
3. 로그인 페이지에서 가입 세션 만료 안내 메시지를 표시한다
4. 신규 회원의 약관 동의 → 회원가입 → 로그인 흐름을 다시 점검한다
