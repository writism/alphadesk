## 1. "Alpha"의 금융적 의미와 서비스 방향

금융에서 **Alpha(α)** = 시장 대비 초과 수익.  
단순히 시장을 따라가는 것이 아니라, **남들보다 먼저 기회를 포착하는 것**을 의미한다.

> **Alpha Desk = "AI가 먼저 포착한 알파 기회를 전달하는 전문 트레이딩 워크스테이션"**

이 방향성을 서비스 설계의 기준으로 삼으면 이름값을 제대로 살릴 수 있다.

### 1-1. "Desk"에 어울리는 추가 프레이밍 (보강)

이름이 주는 이미지는 **한 자리에서 판단·추적·실행(또는 다음 행동)**이 모이는 **데스크**다. Alpha만 강조하면 "예측 앱"으로 좁혀 보이기 쉬우니, 아래 축을 **같은 홈 안에서 짧은 레이어**로 쌓는 것을 권장한다.

| 레이어 | 질문 | 홈에서의 역할 |
|--------|------|----------------|
| 시장 | 장은 어디쯤인가? | 지수·변동성·주도 섹터 한 줄 |
| 나 | 내 관심은 어디에 걸려 있는가? | 관심 요약, 이벤트 임박, 집중도 |
| AI | 우리 모델은 무엇을 말하는가? | 감성·스코어·적중률(정의 명시) |

- **벤치마크 대비**: 코스피/코스닥·대표 ETF 대비 관심 바스켓의 상대 성과·변동성을 한 줄로 두면 "알파"와 직결된다.
- **이벤트 데스크**: 실적·금리·CPI·배당락·공시 등 **일정과 내 종목 교차**는 프로 데스크 감성에 잘 맞는다.
- **리스크 데스크**: 상위 종목 편중, 섹터 편중, 실적 임박 종목 비중 등 **알파를 지키는** 지표는 신뢰와 차별화에 도움이 된다.

## 2. 홈 화면 아이디어

### 2-1. AI 예측 적중률 트래커 ⭐ (핵심 제안)

우리 AI가 **얼마나 맞췄는지** 보여주는 것 자체가 신뢰도이자 차별화 포인트.  
홈 화면에서 첫눈에 보이면 임팩트가 크다.

```
┌─────────────────────────────────┐
│  AI 예측 적중률  (최근 30일)      │
│                                 │
│  ████████░░  73%               │
│  분석 142건 중 104건 적중         │
│                                 │
│  POSITIVE 예측 → 실제 상승 68%   │
│  NEGATIVE 예측 → 실제 하락 79%   │
└─────────────────────────────────┘
```

**필요 데이터:**

- 과거 AI 감성 예측 결과 (`sentiment`, `sentiment_score`)
- 예측 이후 N일간 실제 주가 등락
- 적중 기준 정의 (예: 예측 후 3거래일 내 방향 일치)

**보강 — 정의와 투명성:**

- 적중률은 **정의에 따라 크게 달라진다**. UI에 **기준(N거래일, 방향 정의, 표본 수)**을 항상 노출하는 것이 신뢰에 유리하다.
- **생존자 편향**·**표본 편향**(분석한 종목만 집계)을 문서/툴팁에 짧게 고지할 것.

---

### 2-2. 시장 전체 흐름 + AI 감성 비교 차트

관심종목들의 **AI 감성점수 평균 vs 실제 지수 흐름** 오버레이.  
"우리 AI가 시장을 선행했는가"를 한눈에 확인 가능.

```
        KOSPI 실제등락  ━━━
        AI 감성 종합    ┅┅┅

  +2% │     /\  ┅┅┅┅
      │    /  \/     \
   0% │───/──────────\──
      │  /            \━
  -2% │
      └──────────────────
        1주  2주  3주  4주
```

**필요 데이터:**

- 날짜별 AI 감성점수 평균 (기존 분석 로그 집계)
- KOSPI/KOSDAQ 일별 지수 (외부 API 연동)

**보강 — 모바일·가독성:**

- 모바일 홈에서는 **전체 폭 차트**보다 **스파크라인 + 범례 한 줄 + 기간 선택**이 읽기 쉬운 경우가 많다.
- Y축 스케일(지수 수익률 vs 정규화된 감성)이 다르면 **이중축 또는 정규화 기준**을 라벨로 명시한다.

---

### 2-3. 시장 센티멘트 게이지

CNN Fear & Greed Index처럼, **우리 AI 기반 자체 시장 온도 지수** 산출.  
관심종목 감성점수 집계 → 시장 전반 분위기 수치화.

```
  공포 ◀──────●──────▶ 탐욕
         현재: 61 (중립~탐욕)

  관심종목 평균 감성: +0.34
  상승 예측 종목: 7개 / 하락 예측: 3개
```

**산출 방식 (안):**

```
센티멘트 지수 = (관심종목 평균 sentiment_score + 1) / 2 × 100
→ 0~100 범위, 50 기준 탐욕/공포 구분
```

**보강:**

- "시장 전체"가 아니라 **내 관심 기반 온도**임을 라벨로 분명히 한다 (오해 방지).

---

### 2-4. 홈 화면 전체 레이아웃 (제안)

```
┌──────────────────────────────────┐
│  📈 KOSPI 2,847  +0.3%           │  ← 실시간 지수 티커 바
│  📉 KOSDAQ 832  -0.1%            │
├──────────────────────────────────┤
│  🤖 AI 예측 적중률  73%           │  ← 신뢰도 지표 (핵심)
│  시장 센티멘트  ████░  탐욕 61    │  ← 감성 게이지
├──────────────────────────────────┤
│  ⚡ 오늘의 알파 기회               │  ← AI Top Picks
│  NVDA  알파스코어 88  AI↑ 실제↑ ✓ │
│  TSLA  알파스코어 71  AI↑         │
│  KAKAO 알파스코어 65  AI↓         │
├──────────────────────────────────┤
│  📋 관심종목 요약                  │  ← 기존 Watchlist 연동
└──────────────────────────────────┘
```

**보강 — 레이아웃에 넣을 만한 블록 (선택):**

- **오늘의 브리핑 (3줄)**: 파이프라인·뉴스·감성을 요약한 카피 + 대시보드/관심으로의 CTA.
- **이번 주 이벤트**: 내 관심 종목의 실적·지표 일정 1~3건.
- **파이프라인/갱신 배지**: "새 분석 N건" — 기존 기능과 자연스럽게 연결.

---

### 2-5. 신뢰·규제 고지 (홈에 반드시 고려)

- **투자 권유 아님**: 점수·적중률·차트는 **참고용 정보**임을 홈 하단 또는 각 블록 툴팁에 고정.
- **모델 메타데이터**: 모델/룰 버전, 집계 시각 — "언제 기준인지"를 보여 주면 신뢰가 쌓인다.
- **불확실성**: 가능하면 방향만이 아니라 **구간·신뢰도(confidence)** 를 같이 노출하는 방안 검토.

## 3. 신규 기능 아이디어

### 3-1. 알파 스코어 (Alpha Score)

각 종목에 **시장 대비 초과수익 가능성을 0~100점으로 점수화**.  
이것이 "Alpha Desk"라는 이름의 핵심 기능이 될 수 있다.

| 점수 범위 | 의미 | 색상 |
|-----------|------|------|
| 80 ~ 100  | 강한 매수 시그널 | 진한 초록 |
| 60 ~ 79   | 매수 관심 | 연초록 |
| 40 ~ 59   | 중립 | 회색 |
| 20 ~ 39   | 주의 | 주황 |
| 0 ~ 19    | 회피 시그널 | 빨강 |

**산출 요소 (안):**

- AI 감성점수 (`sentiment_score`) × 가중치
- 최근 N일 등락 히트맵 상승 비율
- 분석 신뢰도 (`confidence`)
- 거래량 변화율 (미래 연동)

**보강 — 표현 주의:**

- UI 카피는 **"매수/회피 시그널"**보다 **"모델 기반 관심도/위험도"**처럼 중립적 표현을 병행하면 규제·오해 리스크를 줄일 수 있다.

---

### 3-2. 오늘의 알파 기회 (Top Picks)

AI가 자동으로 **지금 가장 주목할 종목 3~5개** 선별해서 홈에 노출.

선별 기준 (안):

- 알파 스코어 상위
- 최근 24시간 이내 분석 완료
- POSITIVE 또는 STRONG_POSITIVE 감성
- 정합 뱃지 MATCH (AI 예측 방향 = 실제 등락 방향 일치)

---

### 3-3. AI 예측 vs 실제 성과 히스토리 (종목별)

종목별로 "AI가 언제 뭐라고 했고, 이후 어떻게 됐는지" 타임라인.

```
NVDA 예측 히스토리
─────────────────────────────────
2026-03-20  POSITIVE (+0.72)  → 3일 후 +4.2% ✓
2026-03-10  NEUTRAL  (+0.08)  → 3일 후 -0.3% —
2026-02-28  NEGATIVE (-0.65)  → 3일 후 -5.1% ✓
```

- **신뢰도 쌓기** + 사용자 학습 도구
- 백테스팅 개념으로 활용 가능

---

### 3-4. 섹터 히트맵 (Sector Sentiment Map)

종목 단위를 넘어 **섹터 단위 AI 감성 집계**.

```
  반도체 🟥  바이오 🟩  금융 🟨
  에너지 🟩  IT     🟥  소비재 🟨
  자동차 🟩  화학   🟨  통신   🟥
```

- 어떤 섹터에 AI 긍정/부정 시그널이 집중되는지 파악
- 종목 선택 전 "어느 섹터를 봐야 하는가" 판단 보조

---

### 3-5. 시그널 알림 센터

AI 분석 완료 시 **푸시 알림 + 홈 배지**.

- "관심종목 NVDA에 새 AI 분석 완료 — 알파스코어 88 🟢"
- 알림 설정: 알파스코어 임계값 지정 (예: 80 이상만 알림)
- 홈 화면 "새 시그널 N건" 뱃지

---

### 3-6. 벤치마크 대비 요약 (관심 vs 지수)

**내 관심(또는 선택 바스켓) vs KOSPI/KOSDAQ/섹터 ETF**의 기간 수익률·변동성·최대 낙폭을 한 카드로 요약.

- Alpha Desk 네이밍과 직결
- 홈 상단 또는 관심 블록 바로 위에 두기 좋음

---

### 3-7. 이벤트·캘린더 데스크

실적 발표, 배당락, 주요 거시 지표 일정을 **내 관심 종목과 교차**해 표시.

- 홈: "이번 주 내 일정 3건" + 전체 캘린더로 이동
- 백엔드: 종목 메타·외부 캘린더 데이터 소스 필요 (단계적 도입)

---

### 3-8. 리스크·집중도 (Concentration)

상위 5종목 비중, 섹터 편중, 단일 이벤트(실적) 임박 노출 비중 등 **집중도 경고**.

- 투자 조언이 아니라 **정보 요약**으로 포지셔닝

---

### 3-9. 시나리오·What-if (경량)

프리셋(금리↑, 환율↑, 유가↑ 등) 선택 시 **과거 구간 유사 패턴 또는 상관 요약**만 보여 주는 경량 버전.

- 무거운 백테스트 대신 **교육·맥락**용으로 홈/대시보드 보조 카드로 시작 가능

## 4. 우선순위 제안

| 우선순위 | 기능 | 이유 |
|----------|------|------|
| ★★★ | AI 예측 적중률 트래커 | 서비스 신뢰도의 핵심 지표, 기존 데이터로 구현 가능 (정의·고지 필수) |
| ★★★ | 알파 스코어 | 서비스 정체성, 기존 sentiment_score + heatmap으로 산출 가능 |
| ★★☆ | 시장 센티멘트 게이지 | 홈 임팩트, 집계 단순 — **"내 관심 기반"** 라벨 권장 |
| ★★☆ | 오늘의 알파 기회 (Top Picks) | 진입점, 알파 스코어 구현 후 파생 |
| ★★☆ | 벤치마크 대비 요약 | 이름·스토리와 직결, 데이터만 확보되면 홈 한 카드로 강함 |
| ★★☆ | 신뢰·규제 고지 + 모델 메타데이터 | 기능이 아니라 **출시 품질**에 필수에 가까움 |
| ★★☆ | 오늘의 브리핑 (3줄) | 파이프라인·뉴스와 연동 시 홈 체류·CTA에 유리 |
| ★☆☆ | 섹터 히트맵 | 종목 메타데이터 섹터 분류 필요 |
| ★☆☆ | AI 예측 vs 실제 히스토리 | 주가 이력·적중 정의 연동 필요 |
| ★☆☆ | 이벤트 캘린더 | 외부/메타 데이터 의존 |
| ★☆☆ | 시나리오·What-if | 초기에는 카피·프리셋 수준으로 시작 가능 |

## 6. 열린 질문 (다음 스프린트에서 결정하면 좋은 것)

1. **적중률의 단일 정의**: N거래일, 방향 임계값, 어떤 가격(종가/시가) 기준인지 제품·백엔드 합의.
2. **적중률 공개 범위**: 전 사용자 집계 vs 내 관심만 — 신뢰 vs 프라이버시.
3. **지수·거시 데이터 소스**: 라이선스·지연시간·캐시 정책.
4. **홈의 1차 타깃**: 신규 유입(시장 맥락) vs 재방문(내 관심·갱신) 비중에 따라 블록 순서 조정.
