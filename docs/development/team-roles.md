## 협업 규칙

- 기능별 담당 분담 — 백엔드/프론트 구분 없이 한 기능 끝까지 담당
- 브랜치: `feature/{이름}-{기능명}` → PR → 팀장 검토 후 머지
- PR: 본인 fork → `EDDI-RobotAcademy/alpha-terminal-ai-server`
- `/implement` 사용 시 강의 예제 그대로 X → Alpha-Desk 도메인에 맞게 구현
- 모르는 부분은 팀장 또는 이승욱님께 질문

## 4주 로드맵 및 담당 배분

### 0단계 — 용어 설명 기능 (Week 1, 병행)

> 어려운 주식 용어를 AI가 쉽게 설명해주는 기능

| 담당   | 작업 내용 |
|--------|-----------|
| 김동녁 | BE `POST /market-analysis/explain-term` API 구현 |
|        | BE LangChain 기반 용어 설명 체인 구현 |
|        | BE 주요 금융 용어 사전 데이터 구축 (PER, EPS, 유동비율 등) |
|        | FE 뉴스/공시 카드에서 용어 클릭 시 설명 팝업 UI |

---

### 1단계 — 데이터 축적 (Week 1~2)

> 사용자 상호작용 데이터를 AI가 이해하는 형식으로 저장

| 담당   | 작업 내용 |
|--------|-----------|
| 김동녁 | `user_profile` 도메인 설계 + 헥사고날 구조 세팅 ✅ |
| 최승호 | BE `GET /users/{user_id}/profile` API 구현 |
|        | FE `/profile` 페이지 — 관심종목 + 상호작용 이력 카드 렌더링, 프로필 수정 |
| 이하연 | BE 자정 배치 프로필 자동 업데이트 스케줄러 구현 |

---

### 2단계 — 지능형 필터링 (Week 2~3)

> AI가 요약 전에 사용자 취향을 먼저 조회해서 맞춤형 분석 제공

| 담당   | 작업 내용 |
|--------|-----------|
| 이승욱 | BE `market_analysis` 체인에 `get_user_profile` Tool 추가 |
|        | BE 사용자 취향 기반 맞춤 프롬프트 개선 |
|        | FE 분석 결과 카드에 개인화 태그 표시 |

---

### 3단계 — 능동형 추천 (Week 3~4)

> 사용자가 앱을 켜지 않아도 AI가 관심 종목 이슈를 먼저 감지해서 알림 전송

| 담당   | 작업 내용 |
|--------|-----------|
| 김동녁 | BE `agent_proactive_recommendation` 도메인 + 백그라운드 에이전트 스케줄러 |
| 김민식 | BE 알림 서비스 (`notifications` 테이블 + CRUD API) |
|        | FE 헤더 알림 아이콘 + 읽지 않은 뱃지, 인박스 드롭다운, 토스트 팝업 |
