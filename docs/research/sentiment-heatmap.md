## 2. 핵심 질문 — 무엇을 보여줄 것인가?

> AI가 "긍정"이라고 예측했는데, 실제 주가는 올랐는가 내렸는가?

### 계산 방법

```
sentimentDirection =
  score > +0.1  → UP
  score < -0.1  → DOWN
  else          → NEUTRAL

priceDirection =
  up / (up + down) > 0.55  → UP
  up / (up + down) < 0.45  → DOWN
  else                      → NEUTRAL

match =
  sentimentDirection === priceDirection  → MATCH (일치)
  둘 중 하나가 NEUTRAL               → UNCLEAR (불분명)
  그 외                              → MISMATCH (불일치)
```

### 예시

| 종목 | 감성 | 실제 등락 | 결과 |
|---|---|---|---|
| 삼성전자 | +0.75 (긍정) | ↑18 ↓8 → 상승 우위 | ✅ 일치 |
| LG화학 | -0.68 (부정) | ↑10 ↓18 → 하락 우위 | ✅ 일치 |
| 네이버 | -0.05 (중립) | ↑12 ↓12 → 보합 | ⬜ 불분명 |
| SK하이닉스 | +0.82 (긍정) | ↑8 ↓16 → 하락 우위 | ❌ 불일치 |

## 3. UI 제안 (3안)

---

### 🅐 안: 예측 정합 뱃지 (Prediction Match Badge)

**위치:** 히트맵 헤더 오른쪽 (가장 심플)

```
┌ 일별 등락 ──────────────── [AI↑ · 실제↑ ✓] ─┐
│ ■■□□■■■□■■□□■■■□■■□□■■■□■■■□□■           │
│ ↑18  ↓8  →4                                  │
└───────────────────────────────────────────────┘
```

**뱃지 3종:**
- `AI↑ · 실제↑ ✓` — 초록 배경 (일치)
- `AI↓ · 실제↓ ✓` — 초록 배경 (일치)
- `AI↑ · 실제↓ ✗` — 빨간 배경 (불일치)
- `AI↓ · 실제↑ ✗` — 빨간 배경 (불일치)
- `─` — 회색 배경 (불분명/중립)

**장점:**
- 기존 컴포넌트 변경 최소화 (`StockDailyReturnsHeatmap`에 prop 1개 추가)
- 한 눈에 파악 가능 (텍스트 + 색상)
- 구현 공수 최소 (1~2시간)

**단점:**
- 감성점수의 강도(크기)는 표현 안 됨 (방향만)

---

### 🅑 안: 이중 방향 게이지 (Dual Direction Gauge)

**위치:** 히트맵과 신뢰도 바 사이 (새 섹션)

```
감성 예측    실제 등락
   ──↑──       ──↑──
   0.75       +33%
   [▓▓▓░]    [▓▓░░]
```

또는 한 줄로:

```
예측 [━━━━━▶] +0.75   실제 [━━▶    ] +33% 상승우위
                            ↕ 방향 일치 ✓
```

**장점:**
- 감성점수의 크기(강도)와 실제 등락 비율을 동시에 비교
- 정량적 비교가 가능 (강한 예측인데 약하게 맞았는지 등)

**단점:**
- 카드에 새 섹션 추가 → 카드 높이 증가
- 구현 공수 중간 (4~6시간)
- 비슷한 정보(신뢰도 바)와 중복될 수 있음

---

### 🅒 안: 히트맵 오버레이 마커 (Heatmap Overlay Marker)

**위치:** 히트맵 시리즈 위에 분석 날짜 기준선 표시

```
┌ 일별 등락 ─────────────────── AI분석일 ─┐
│ ■■□□■■■□■■□□ | ■■□□■■■□■■□□■       │
│              ↑                           │
│         분석 시점                         │
│    이전(실적)    이후(예측 검증 구간)       │
└──────────────────────────────────────────┘
```

분석 날짜를 기준으로 히트맵을 **이전/이후**로 분리해서 보여줌.
- 이전: 분석 근거 구간 (어두운 테두리)
- 이후: 예측 검증 구간 (발광 테두리, 상승/하락에 따라 색상)

**장점:**
- 시계열 맥락에서 예측과 결과를 동시에 볼 수 있음
- 가장 많은 정보를 담음
- 스토리텔링이 강함 ("이 시점에 예측했고, 이후 이렇게 됐다")

**단점:**
- "분석 날짜" 데이터가 현재 카드 props에 없음 (추가 필요)
- 구현 공수 높음 (8~12시간)
- 히트맵이 이미 작아서 분할이 복잡해질 수 있음

## 5. 추천

### 단기: **🅐 안 (예측 정합 뱃지)** 즉시 적용

이유:
- 현재 `StockSummaryCard`에 `sentiment_score`와 `heatmap.summary`가 이미 함께 있음 → 계산 로직만 추가
- 새 백엔드 API 불필요, 새 데이터 구조 불필요
- "심플하게" 요구사항에 가장 부합

### 장기: **🅒 안 (오버레이 마커)** 검토

이유:
- 분석 날짜가 파이프라인 실행 기록에 이미 존재 (`analysis_log`)
- 히트맵에 "이 예측이 맞았는지"를 시각적으로 바로 보여주는 가장 직관적 방법
- 단, 별도 백로그로 분리하여 설계 필요

## 6. 구현 범위 (🅐 안 기준)

```
변경 파일: StockDailyReturnsHeatmap.tsx
추가 prop: sentimentScore?: number

내부 계산:
  sentimentDir = score > 0.1 ? 'UP' : score < -0.1 ? 'DOWN' : 'NEUTRAL'
  priceRatio = up / (up + down)
  priceDir = ratio > 0.55 ? 'UP' : ratio < 0.45 ? 'DOWN' : 'NEUTRAL'
  match = sentimentDir === priceDir ? 'MATCH' : 'MISMATCH' (NEUTRAL → 'UNCLEAR')

뱃지 렌더:
  MATCH   → "AI{방향} · 실제{방향} ✓" (초록)
  MISMATCH → "AI{방향} · 실제{방향} ✗" (빨강)
  UNCLEAR  → 표시 안 함 (또는 "─" 회색)
```
