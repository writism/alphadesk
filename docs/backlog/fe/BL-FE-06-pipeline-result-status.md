# BL-FE-06

**Backlog Type**
UI Backlog

**Backlog Title**
대시보드 페이지가 파이프라인 실행 결과 상태를 표시한다

**배경**
현재 파이프라인이 실패해도 대시보드에 빈 화면만 표시되고 사용자는 원인을 알 수 없다.
`POST /pipeline/run` 응답에 `processed` 배열로 각 종목별 skipped/reason 정보가 포함된다.

**Success Criteria**
- 파이프라인 실행 중에는 로딩 상태가 표시된다
- 실행 완료 후 종목별 성공/실패/스킵 결과가 표시된다
- skipped=true인 종목은 reason(예: "수집된 기사 없음", "분석 실패")이 표시된다
- 분석 결과가 있는 종목은 정상 카드로 표시된다
- 전체 종목이 실패한 경우 안내 문구가 표시된다

**Todo**
1. 파이프라인 실행 버튼 및 실행 결과 상태 영역 UI를 구현한다
2. 종목별 성공/스킵 결과 목록 UI를 구현한다
3. skipped 항목에 reason 텍스트를 표시하는 UI를 구현한다
4. 전체 실패 시 안내 문구 UI를 구현한다
5. 로딩 및 에러 상태 UI를 구현한다
