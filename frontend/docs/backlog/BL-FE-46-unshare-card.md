# BL-FE-46 공유 카드 삭제 (공유 취소)

## 배경
현재 사용자가 공유한 카드는 홈 피드에 계속 노출되며 직접 내릴 수 있는 방법이 없다.
공유를 취소하고 싶을 때 피드에서 본인 카드만 삭제할 수 있어야 한다.

## 목표
- 로그인한 사용자가 자신이 공유한 카드에만 "공유 취소" 버튼을 표시한다.
- 버튼 클릭 시 확인 후 카드를 피드에서 제거한다.
- 비로그인 사용자 또는 타인의 카드에는 버튼이 노출되지 않는다.

## 구현 범위

### 백엔드 (이미 구현됨)
- `DELETE /card-share/{card_id}` — 본인 카드 여부 검증 후 삭제 (HTTP 403 처리 포함)

### 프론트엔드
- `SharedCard` 도메인 모델에 `sharer_account_id` 추가
- `SharedCardResponse` 백엔드 응답에 `sharer_account_id` 포함
- 로그인 사용자 account_id를 `account_id` 쿠키에서 읽어 비교
- `SharedCardFeed` → `SharedCardItem`: 본인 카드일 때 ··· 메뉴 또는 삭제 버튼 표시
- 삭제 클릭 → 확인 dialog → `DELETE /card-share/{id}` 호출 → 피드 즉시 갱신 (`reload`)

## 완료 기준
- 로그인 사용자가 자신의 카드에서만 삭제 버튼이 보인다
- 삭제 후 피드에서 즉시 사라진다
- 타인의 카드 또는 비로그인 상태에서는 버튼이 없다
