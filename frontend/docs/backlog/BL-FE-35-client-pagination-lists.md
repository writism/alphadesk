# BL-FE-35

**Backlog Type**  
UX / Performance

**Backlog Title**  
관심종목 목록·대시보드 최근 AI 분석 로그 — 클라이언트 페이지네이션

---

## 1. 배경

- 관심종목·분석 로그는 현재 API가 **전체 배열**을 내려주고, 프론트가 한꺼번에 렌더한다.
- 항목이 수십 개 이상이면 **스크롤·DOM·히트맵 셀** 부담이 커진다.

## 2. 목표 (Success Criteria)

| ID | 기준 |
|----|------|
| SC-1 | **관심종목** 목록은 기본 **페이지당 N건**(예: 10)씩만 렌더한다. |
| SC-2 | **최근 AI 분석 로그**(펼친 상태)도 동일 패턴으로 페이지당 N건씩 렌더한다. |
| SC-3 | **이전 / 다음**(또는 동등 접근성)으로 페이지를 바꿀 수 있고, **현재 구간·전체 건수**를 표시한다. |
| SC-4 | 히트맵 배치 API는 **전체 심볼** 기준으로 유지(한 번 fetch); 페이지는 **표시만** 나눈다. |
| SC-5 | 한 페이지 이하이면 페이지네이션 UI는 **숨긴다**. |

## 3. 비범위 (후속)

- 서버 사이드 페이지네이션·`/pipeline/logs?page=` 등 API 변경.
- 페이지 크기 사용자 설정.
