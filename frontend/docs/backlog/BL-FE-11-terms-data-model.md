# BL-FE-11

**Backlog Type**
Behavior Backlog

**Backlog Title**
애플리케이션이 약관 목록을 Frontend 내부 컴포넌트에서 조회한다

**배경**
기존 약관 페이지는 TERMS_ITEMS 상수를 page.tsx 내부에 직접 정의하여 도메인 모델과 UI가 결합되어 있다.
약관 데이터를 도메인 모델로 분리하고, 정적 데이터로 구조화하여 재사용 가능하게 만들어야 한다.
외부 API 호출 없이 Frontend 내부에서 조회한다.

**Success Criteria**
- 약관 동의 페이지 진입 시 약관 목록 데이터를 조회한다
- 약관 목록에 이용약관, 개인정보처리방침, 아동 보호 정책 항목이 포함된다
- 각 약관 항목은 TermSection[] 구조로 제목과 내용 배열을 가진다
- 각 약관 항목에 필수/선택 여부 정보가 포함된다
- 약관 데이터가 Frontend 내부에 정적으로 정의되어 외부 API 호출 없이 조회된다

**Todo**
1. TermSection, TermItem 도메인 모델을 정의한다
2. 이용약관, 개인정보처리방침, 아동 보호 정책 정적 데이터를 구성한다
3. 각 약관 항목에 필수/선택 여부를 지정한다
4. 약관 목록을 조회하는 useTerms 훅을 구현한다
